## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
CLAUDE.md L0 Config declares worktree-symlinks: node_modules — the project's own worktree-creation path (plugin/scripts/handle-basic-ready.sh using git worktree add) relies on this convention being honored so a fresh task worktree gets a working node_modules without a full bun i. Note (audit 2026-07-14): the originally-cited trigger — the generic Agent tool's isolation:'worktree' option — is external harness behavior outside this repo's control and cannot be fixed here. The actionable, in-repo scope is limited to this project's own worktree-creation code paths (handle-basic-ready.sh and any other script/engine code that runs 'git worktree add' for a task), which must symlink node_modules themselves so they don't depend on the caller (Agent tool or otherwise) doing it. This silently broke src/test/epicd-plugin-synthetic-repo.test.ts's bun run build step (relative ./node_modules/@tailwindcss/cli/... path) when the convention wasn't honored, which in turn made every bun test DoD gate fail, which in turn made completeTask (src/engine/complete.ts) skip the merge entirely and route to needs-human — with no error message pointing at the real cause. Discovered and worked around manually (ln -s <main-repo>/node_modules node_modules) during BACK-687; this task is to fix the gap at the source in this repo's own worktree-creation code, rather than repeat the workaround per ad hoc worktree.
