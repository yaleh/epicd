## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
CLAUDE.md L0 Config declares worktree-symlinks: node_modules — the project's own worktree-creation path (plugin/scripts/handle-basic-ready.sh using git worktree add) relies on this convention being honored so a fresh task worktree gets a working node_modules without a full bun i. The generic Agent tool's isolation:'worktree' option (used ad hoc during BACK-687's fixpoint-convergence drive) creates a plain git worktree with no node_modules at all — not even a broken symlink. This silently broke src/test/epicd-plugin-synthetic-repo.test.ts's bun run build step (relative ./node_modules/@tailwindcss/cli/... path), which in turn made every bun test DoD gate fail, which in turn made completeTask (src/engine/complete.ts) skip the merge entirely and route to needs-human — with no error message pointing at the real cause. Discovered and worked around manually (ln -s <main-repo>/node_modules node_modules) during BACK-687; this task is to fix the gap at the source rather than repeat the workaround per ad hoc worktree.
<!-- SECTION:DESCRIPTION:END -->
