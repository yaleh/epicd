## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
Implemented both parts.

Part 1 (inline gate-review, AC#3): TaskList.tsx renders approve/reject/escalate buttons on rows where getPhaseActor(pipeline_id, phase)==="human". New pure helpers in driver-indicator.ts: computeApprovePhase(pipelineId, phase) (forward-search-then-wrap to the next actor=machine phase in the same pipeline; e.g. execution/needs-human -> "ready") and ESCALATE_TRANSITION (fixed {pipeline_id:"execution", phase:"needs-human"}, since that's the one phase the spec names explicitly and it lets any human-actor row, e.g. authoring/backlog, escalate into execution's human queue). Reject reuses the existing archiveTask (DELETE /api/tasks/:id) mechanism — no new phase name invented. Write-back goes through apiClient.updateTask -> PUT /api/tasks/:id; server/index.ts's handleUpdateTask now forwards pipeline_id/phase into TaskUpdateInput (core.updateTaskFromInput already supported both, BACK-610). No new WS code needed: core's file write is picked up by the existing ContentStore watcher -> broadcastTasksUpdated -> tasks-updated, already wired since BACK-644/645.

Part 2 (status-label convergence, AC#5): new src/web/lib/status-label.ts with label(phase, actor) — a pure phase+actor -> Tailwind color-class projection reading only pipeline-data (getPhaseActor), never per-task-only heuristics — plus getStatusBadgeClass(status, phase?, pipelineId?) which uses label() when phase/actor resolve and otherwise falls back to a status-substring heuristic equivalent to the union of the 3 pre-convergence implementations (preserves legacy/no-phase behavior). Converged TaskList.tsx's getStatusColor, TaskColumn.tsx's getStatusBadgeClass (passes one representative task's phase/pipeline_id alongside the column's status string), and MilestonesPage.tsx's getStatusBadgeClass (passes task.phase/task.pipeline_id) to call this one function. Left terminal-status.ts untouched (different concern, out of scope per task description).

Tests added: src/web/lib/driver-indicator.test.ts (computeApprovePhase, ESCALATE_TRANSITION) and new src/web/lib/status-label.test.ts (label() truth table + getStatusBadgeClass fallback cases).

Gates green: bunx tsc --noEmit clean; bun run check . clean (0 errors, pre-existing warnings only in files I didn't touch); bun test --parallel: 1882 pass / 0 fail / 2 skip (236 files).
